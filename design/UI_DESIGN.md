# Jnote UI 设计提案

## 设计定位

Jnote 是面向开发者与知识整理场景的桌面端笔记软件。设计关键词是：轻量、清醒、秩序感、带一点新锐的色彩锋芒。它不做浮夸的卡片堆叠，也不做大面积渐变装饰，而是把主要工作区留给文件、编辑和阅读。

## 核心布局

主界面采用四层结构：

1. 顶部工具栏：保留文件操作、查找、保存、撤销重做、插入图片、侧边栏开关等常用入口。
2. 页签书签栏：展示所有已打开文件，当前文件高亮；每个书签有关闭按钮，长文件名通过悬停展示；页签过多时横向滚动。
3. 右侧堆叠文件树：允许多个顶级目录同时固定展示，默认最多保留十个最近打开目录；不同格式文件使用不同底色标记。
4. 中央 Markdown 工作区：采用单一工作状态，直接以结构化 Markdown 内容作为主工作面；一级标题使用固定色轮播背景，标题带折叠按钮；代码块使用类似 IDE 的多色语法主题。

已移除“大纲与状态”辅助栏和文档信息/视图切换栏，避免重复信息干扰编辑。界面只保留核心工作流：打开文件、管理目录、编辑内容。

## 视觉系统

### 配色

- 背景：偏暖的浅灰白，降低长时间阅读疲劳。
- 主文字：近黑墨色，保证编辑器可读性。
- 强调色：青绿、珊瑚、靛蓝、金黄轮播，用于 Markdown 一级标题区分。
- 文件类型底色：
  - txt：柔和蓝灰
  - word：淡蓝
  - markdown：淡绿
  - 图片目录：淡橙

### 字体

- UI 字体：优先使用 `Microsoft YaHei UI`、`Segoe UI`。
- 编辑器正文：中文使用系统 UI 字体，代码使用 `JetBrains Mono`、`Consolas`。
- 字号体系控制在 12 到 24px 之间，保持桌面软件应有的信息密度。

### 按钮

- 工具栏以图标按钮为主，文字按钮只用于明确的主操作。
- 按钮圆角控制在 8px 以内。
- 悬停状态使用轻微底色和阴影，不做夸张动画。

## 功能映射

| 功能 | UI 位置 |
| --- | --- |
| 新建文件 `Ctrl+N` | 顶部工具栏图标按钮 |
| 新建文件夹 `Ctrl+Shift+N` | 顶部工具栏图标按钮、文件树目录操作 |
| 查找 `Ctrl+F` | 顶部工具栏搜索框 |
| 撤销/重做 `Ctrl+Z / Ctrl+Y` | 顶部工具栏图标按钮 |
| 收起/展开侧边栏 `Ctrl+B` | 顶部工具栏图标按钮 |
| 打开网页链接 `Ctrl+鼠标左键` | 编辑器链接样式保留 |
| 插入图片 | 顶部工具栏图标按钮 |
| 双击查看图片 | 编辑区图片附件缩略图 |
| 记录上次打开文件 | 页签栏和右侧最近目录区域 |
| 记录 Markdown 折叠状态 | 标题折叠按钮与页签状态记录 |

## Markdown 展示规则

- 一级标题：按一级标题生成整块内容区域，标题和其下方正文、列表、代码块共用同一底色，颜色按固定数组轮播。
- 二级标题：更窄的浅色标记条，附属于一级标题区域。
- 标题折叠：标题左侧使用折叠图标；折叠后保留标题行和行数提示。
- 无序列表：保留 `- ` 语义，但渲染为轻量项目符号。
- 代码块：显示语言标签；Java、Python、SQL 等使用关键字、字符串、注释、类型、函数不同颜色。

## 原型文件

高保真静态原型位于：

- `design/jnote-ui-prototype.html`

该原型用于审核视觉设计与布局，不代表最终 Java 功能已经完成。确认设计方向后，再进入 Java 桌面端实现阶段。
